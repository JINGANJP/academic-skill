# Academic Skill

这是一个面向学术研究与论文写作场景的 Codex Skill。它不是单纯的“论文润色器”或“文案生成器”，而是一个强调用户对齐、证据充分和按任务体量执行的学术代理：先明确目标，再按需要学习领域、筛选文献、验证方向、补强数据与证据，最后再进入正式写作。

## 功能概览

- 支持多语言学术场景，包括中文、英文和中英混合任务
- 支持按学科触发与工作，例如经济学、计算机科学、医学、公共卫生、社会科学等
- 在写作前优先进行领域学习，区分经典文献与前沿研究
- 自动过滤低质量、弱相关或噪声来源，优先保留可信与高价值材料
- 可在需要时生成或验证研究 idea，并从创新性、数据可得性、实验可行性、代码复现难度等维度进行筛选
- 在用户选定方向后，整理真实参考文献、数据路径、证据包与写作提纲
- 对硕士论文等大体量任务采用拆分写作与逐段验收，而不是一次性整篇生成
- 当数据不足且任务确实需要时，可先确认规模需求，再进行数据采集、清洗与记录
- 要求引用真实有效，不允许伪造文献、作者、年份、DOI、数据集或实验结论

## 适用场景

这个 skill 适合以下任务：

- 论文选题与研究 idea 设计
- 文献综述与 related work 写作
- 开题报告、研究计划与 proposal 撰写
- 本科、硕士、博士论文写作
- 中文 CSSCI / 中文核心风格论文写作
- 英文学术论文结构规划与初稿撰写
- 研究方法、实验路径、数据来源与可行性评估

## 核心流程

这个 skill 的默认工作逻辑现在更轻、更强调用户参与：

1. 先对齐任务目标、交付形式、语言与字数要求
2. 根据任务体量选择轻量支持、标准项目或长论文模式
3. 学习当前任务真正需要的领域内容，并建立“足够用”的文献基础
4. 在需要时验证选题、方法、实验设计或数据路径
5. 在证据不足时补强参考文献、数据与方法，而不是直接堆砌文案
6. 对长任务进行章节或模块拆分，逐段写作、检查和汇总

## 中文写作支持

这个 skill 对中文语境做了专门优化。

- 共享中文学术表达规则：避免英文化直译和空泛评价
- 中文论文版风格：更适合本科、硕博论文、章节型长文、开题报告
- 中文期刊版风格：更适合 CSSCI、中文核心、投稿型文章，表达更紧凑，问题意识更强

## 使用方式

你可以在需要时显式加入这个 skill，例如：

```text
Use $academic-research to align the task, choose the lightest suitable workflow, learn the economics literature on digital finance, validate feasible directions, and draft a Chinese thesis-style literature review.
```

或者中文使用：

```text
使用 $academic-research 帮我先对齐任务目标，再学习“平台经济中的算法定价”这一方向，筛选高质量文献，验证可行研究方向，并按中文期刊风格准备写作提纲。
```

## 目录结构

仓库中的核心内容包括：

- `academic-research/SKILL.md`
  主 skill 说明文件，定义触发条件、工作流程与约束规则
- `academic-research/agents/openai.yaml`
  skill 的展示元数据
- `academic-research/references/`
  各类参考规则文件，包括领域学习、idea 验证、文献综述、引用规范、中文论文风格、中文期刊风格，以及跨 Agent 迁移说明等

## 设计原则

这个 skill 的设计目标不是“快速写出看起来像论文的文字”，而是尽量让模型在学术任务中表现得更像一个有研究习惯、能尊重用户目标的助手：

- 不跳过前期研究
- 不编造文献和引用
- 不把弱 idea 直接包装成“创新点”
- 不用统一模板强行覆盖所有学科
- 不把中文写作做成英文句式的机械翻译
- 不为小任务强行启动重型流程
- 不用漂亮措辞掩盖字数不足、文献不足或数据薄弱

## 提醒

这个 skill 可以显著提升学术任务的严谨性，但它仍然需要用户对研究问题和最终结论承担学术责任。尤其是在医学、公共卫生、政策研究等高风险领域，仍然应由使用者进一步核验关键资料、证据等级和伦理要求。
